程序员的自我修养

发表于 2022-02-17 分类于 读书笔记
本文字数： 4.2k 阅读时长 ≈ 4 分钟

第一章、职业生涯

年轻程序员的10条建议

  1. 正确地认识自己：找一个真正反映内心的词汇来定位自己
  2. 比一般人更加努力；程序员是一个需要花时间学习新技术的职业，10000个小时的学习实践可以让你精通
  3. 适时建立个人权威：选择时机去表现自己的能力和责任，建立良好的影响力
  4. 遵循最佳实践：理解和学习优秀的框架和软件开发思想
  5. 保持好奇心并乐于探索新的事物：除了自己工作范围，也可以研究更底层的框架实现以及看似与工作无关的技术，会提升原有技能
  6. 抛开代码与人沟通：除了钻研技术外，人际关系、沟通技巧对工作也有很大的帮助
  7. 为优秀的人工作：周围的环境对自身的影响巨大，当周围的人无法让自己获得提升时，尝试改变
  8. 生活、睡眠、旅行：明白生活才是存在的意义，适当放松能更好地调整自己的状态
  9. 相信自己的天赋和创造力：天赋与创造力需要在生活中培养和挖掘，多阅读、记录和动手
  10. 书上没提到的，但我觉得很重要的，选择最适合自己的方式和路线，例如找工作时纠结的算法或开发，在学生时代的尝试基本上可以帮助你确定大方向

后知后觉的职业生涯经验

  1. 薪酬与工作量无关：薪酬最本质取决于你在公司中的可替代性，努力修炼技术让自己变得重要
  2. 尽可能持续做一件事：坚持一个项目尽量保证深度，是提升自己重要性的重要途径
  3. 唯一不变的是变化本身：很哲学的一句话，但也符合现实。最佳途径是提升自己快速学习的能力，对于出现的新技术可以更快地掌握，准备更多
  4. 声誉非常重要：做事严谨，减少出错概率；建立声誉要长时间，但失去一次就够
  5. 理解沟通的意义：除技术外，沟通是另一个重要的技能。只有充分理解用户需求，才可以真正解决问题
  6. 右脑是成功的关键：IT高管除代码外，还有写PPT、演讲、说服力等技能，是在技术的基础上，使自己与众不同
  7. 不要轻易说简单或不可能：草率的回答会给自己减分，尽量使用科学的方法进行有效地表达
  8. 不应该总是单打独斗：要做有影响力的项目，需要团队，融入团队是一个基础，而与他人的沟通也是提升自己的机会
  9. 能力显而易见：优秀的程序员时刻体现自己的能力，从解决技术难题到优雅的代码，这来自于大量的编码实践和持续学习思考

GFS阅读笔记

发表于 2022-02-16 更新于 2022-02-17 分类于 课程笔记
本文字数： 4.9k 阅读时长 ≈ 4 分钟

Google File System是面向大规模数据密集型应用、可扩展的分布式文件系统

特性

  1. 将组件失效认为是常态而不是意外，几百台设备出现任意损坏的几率非常高，所以GFS集成了持续的监控、错误检测、容错以及自动恢复
  2. 文件规模巨大且快速增长，如果出现数亿个小文件，普通的文件系统是笨拙的，需要根据应用特点做对应优化，例如IO操作和Block的大小重新考虑
  3. 大部分文件的修改是在文件尾部追加，而不是覆盖原有数据。首先是应用特点决定大部分文件写入后只需要按顺序读，几乎不存在随机写，所以采用追加的方式更贴合场景
  4. GFS没有遵守POSIX的文件系统规范，而是放松了要求，提高系统的灵活性。引入原子性的记录追加操作，保证多个客户端同时进行追加，而不需要额外的同步操作保证数据的一致性

设计的出发点

  1. 系统在许多廉价机器上运行，机器故障是常态，系统要能快速检测、做冗余和恢复
  2. 系统存储的多为大文件，针对这正特点进行优化，也必须支持小文件
  3. 工作负载主要为：大规模的流式读取和小规模的随机读取。如果比较关注性能，则将小规模的随机读取操作合并、排序，之后顺序批量读取，避免在文件中移动读取位置
  4. 工作负载还包括写操作，通常是大规模的、顺序的、追加方式的
  5. 系统必须高效的、明确的定义实现多客户端并行追加数据到同一个文件里的语意，文件通常被用于“生产者-消费者”队列，或多路文件合并
