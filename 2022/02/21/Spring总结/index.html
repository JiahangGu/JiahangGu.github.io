Spring

部分参考自博客

Spring核心组件

  1. 核心容器：提供Spring框架的基本功能。主要组件是BeanFactory，是工厂模式的实现。BeanFactory使用IoC模式将应用程序的配置和依赖性规范与实际的应用程序代码分开
  2. Spring上下文：一个配置文件，向框架提供上下文信息
  3. Spring AOP：通过配置管理特性，AOP模块直接将面向切面的编程功能集成到了Spring框架中。可以将一些通用任务，如安全、事务、日志等集中进行管理，提高了复用性和管理的便捷性
  4. Spring DAO：为JDBC DAO抽象层提供了异常层次结构，可以用该结构来管理异常处理和不同数据库供应商的错误消息。简化了错误处理，并极大降低了需要编写的异常代码数量
  5. Spring ORM：提供若干个ORM框架，提供了ORM的对象关系工具，包括JDO、Hibernate和Ibatis SQL Map，遵从Spring的通用事务和DAO异常层次结构
  6. Spring Web：Web上下文模块建立在应用程序上下文模块之上，为基于Web的应用程序提供了上下文。
  7. Spring MVC：全功能的构建Web应用程序的MVC实现。容纳了大量的视图技术

常用注解

@Controller, @RestController, @Component, @Repository, @Service
@ResponseBody, @RequestMapping, @Autowired, @PathVariable, @RqeuestParam, @RequestHeader
@ModelAttribute, @SessionAttributes, @Valid, @CookieValue

第三方框架

  1. 权限：Shiro，Spring Security
  2. 缓存：Ehcache，Redis
  3. 持久层框架：Hibernate，Mybatis
  4. 定时任务：quartz，Spring-Task

Spring的配置方式

  1. 基于XML的配置

  3. 基于Java的配置

Spring优点

  1. 低侵入，代码污染低
  2. DI机制将对象之间的依赖关系交由框架处理，减低组件的耦合性
  3. 提供了AOP技术，支持将一些通用任务，如安全、事务、日志、权限等进行集中式管理，更好复用
  4. 对主流应用框架提供集成支持

如何理解IoC

控制反转，指创建对象的控制权的转移，以前创建对象的主动权和时机自己控制，现在转移到Spring容器中，并由容器根据配置文件去创建实例和管理各个实例之间的依赖关系，对象间松散耦合，也利于功能复用。IoC让对象的创建不用new，而是由Spring生产，使用Java的反射机制，根据配置文件在运行时动态地去创建及管理对象，并调用对象方法

IoC的注入方式

  1. 构造器注入：bean设置时利用构造器参数注入
  2. setter方法注入：bean设置时利用property参数构造，即传入set()方法的参数
  3. 静态工厂注入：通过调用静态工厂的方法获取需要的对象。bean设置指定class和factory-method
  4. 实例工厂：获取对象实例的方法表示静态的，需要首先new工厂类，然后调用普通实例方法。bean设置factory-bean和factory-method

如何理解AOP

面向切面编程，是一种面向对象的补充，用于将与业务无关，但对多个对象产生影响的公共行为和逻辑，抽取并封装成一个可重用的模块，即切面。
AOP减少了系统中的重复代码，降低了模块间的耦合度，同时提高了系统可维护性

AOP实现

代理模式。

  1. 静态代理，如AspectJ
  2. 动态代理，JDK动态代理和CGLib动态代理。如果目标类是接口，则使用JDK动态代理，否则使用Cglib。

JDK动态代理：使用java.lang.reflect包中的：Proxy和InvocationHandler。InvocationHandler是一个接口，通过实现该接口定义横切逻辑，并通过反射机制调用目标类的代码，动态将横切逻辑和业务逻辑编制在一起。Proxy利用InvocationHandler动态创建一个复合某一接口的实例，生成目标类的代理对象
CGLib动态代理：可以在运行期扩展Java类和实现Java接口。如果没有定义接口，则只能使用CGLib

AOP主要应用场景

  1. 权限
  2. 缓存
  3. 内容传递
  4. 错误处理
  5. 懒加载
  6. 调试
  7. 日志，跟踪，优化，校准
  8. 性能优化
  9. 持久化
  10. 资源池
  11. 同步
  12. 事务

AOP核心概念

  1. 切面：类是对物体特征的抽象，切面就是对横切关注点的抽象，关注点的模块化
  2. 横切关注点：对哪些方法拦截，拦截后如何处理，这些关注点
  3. 连接点：被拦截到的点（方法），还可以是字段或构造器，比如某方法调用的时候或处理异常的时候，Spring AOP中表示一个方法的执行
  4. 切入点：对连接点进行拦截的定义。通知和一个切入点表达式关联，在满足表达式匹配的连接点上执行
  5. 通知：指拦截到连接点之后要执行的代码。包括前置、后置、异常、最终和环绕通知。
  6. 目标对象：代理的目标对象。被一个或多个切面通知的对象
  7. 织入：将切面应用到目标对象并导致代理对象创建的过程
  8. 引入：不修改代码前提下，在运行期为类动态添加方法或字段
  9. AOP代理：AOP框架创建的对象，用来实现切面契约

AspectJ和Spring AOP的区别

  1. AspectJ是静态代理的增强，静态代理是编译阶段将切面织入到字节码中，运行时就是增强之后的AOP对象
  2. Spring AOP使用的动态代理，AOP框架不修改字节码，而是在每次运行时在内存中临时为方法生成一个AOP对象，包含了目标对象的全部方法，并在特定切点做增强处理，并回调原对象的方法

BeanFactory和ApplicationContext

是Spring的两大核心接口，作为Spring的容器，BeanFactory是Spring最底层的接口，包含各种bean定义，读取bean配置文档，管理bean的加载、实例化，控制bean的生命周期，维护bean之间依赖关系，ApplicationContext接口为BeanFactory的派生，基于BeanFactory的功能提供了更完整的框架功能。

BeanFactory和ApplicationContext的区别

  1. BeanFactory使用延迟加载的形式注入Bean，只有用到某个Bean时才会进行加载实例化，而ApplicationContext会在容器启动时一次性创建所有bean
  2. BeanFactory通常以编程方式创建，而ApplicationContext还能以声明的方式创建，如使用ContextLoader
  3. BeanFactory和ApplicationContext都支持BeanPostProcessor、BeanFactoryPostProcessor的使用，区别是BeanFactory需要手动注册

Spring Bean的生命周期

  1. 实例化Bean：对于BeanFactory容器，客户向容器请求一个尚未初始化的bean时，或初始化bean时需要注入另一个尚未初始化的依赖时，容器调用createBean进行实例化。对于ApplicationContext容器，当容器启动结束后，通过获取BeanDefinition对象中的信息，实例化所有的bean
  2. 设置对象属性：实例化的对象封装在BeanWrapper对象中，Spring根据BeanDefinition中的信息以及通过BeanWrapper提供的设置属性的接口完成依赖注入
  3. 处理Aware接口：Spring检测该对象是否实现了xxxAware接口，并将相关实例注入给Bean。如果bean实现了BeanNameAware接口，会调用实现的setBeanName(String beanId)方法，传递的是Spring配置文件中Bean的id；如果Bean实现了BeanFactoryAware接口，会调用实现的setBeanFactory()方法，传递的Spring工厂自身；如果Bean实现了ApplicationContextAware接口，会调用setApplicationContext(ApplicationContext)方法，传入Spring上下文
  4. InitializingBean与init-method：如果Bean在Spring配置文件中配置了init-method属性，会自动调用其配置的初始化方法
  5. BeanPostProcessor：在Bean初始化结束时调用，如果想对Bean进行一些自定义的处理，可以让Bean实现BeanPostProcessor接口，该步骤结束后，Bean被正确创建
  6. DisposableBean：Bean不再需要时，会经过清理阶段，如果Bean实现了DisposableBean接口会调用其实现的destory()方法
  7. destory-method：如果Bean的Spring配置了destory-method属性，自动调用其配置的销毁方法

bean作用域

  1. singleton：默认，每个容器只有一个bean实例，由BeanFactory自身维护
  2. prototype：为每个bean请求提供一个实例
  3. request：为每个请求创建一个实例，请求完成后被回收
  4. session：为每个session创建一个实例，过期后被回收
  5. global-session：全局作用域

Spring中单例bean是线程安全吗

Spring没有对单例Bean进行多线程的封装处理。如果bean没有可变状态则线程安全，否则需要自行保证线程安全

Spring如何处理线程并发问题

一般只有无状态Bean才可以多线程环境下共享，绝大部分bean都可以声明为singleton作用域，因为Spring对一些bean中非线程安全状态采用ThreadLocal处理，解决线程安全问题

Spring自动装配

xml配置中共有5种自动装配：

  1. no：默认方式不进行自动装配，手工设置ref属性装配
  2. byName：通过bean的名称自动装配
  3. byType：通过参数类型自动装配
  4. constructor：利用构造函数进行装配，并且构造函数的参数通过byType装配
  5. autodetect：自动探测，如果有构造方法，通过construct方式自动装配，否则使用byType方式装配

基于注解的方式：使用@Autowired，需要配置<context:annotation-config />
会在容器查询对应类型的bean，如果一个则装配，如果大于一个根据名称查找，如果为空则抛异常。可以指定required=false

@Autowired和@Resource之间区别

  1. @Autowired默认按照类型装配注入，默认情况下要求依赖对象必须存在
  2. @Resource默认按照名称来装配注入，只有找不到名称匹配的bean才会按照类型装配注入

Spring的设计模式

  1. 工厂模式：BeanFactory就是简单工厂模式的体现，用于创建对象实例
  2. 单例模式：bean默认单例模式
  3. 代理模式：AOP用到了JDK动态代理和CGLib字节码生成
  4. 模板方法：解决代码重复的问题，如RestTemplate
  5. 观察者模式：实现对象键一对多的依赖关系，当一个对象状态改变时，所有依赖的对象都会得到通知并更新，如listener

Spring事务的种类及区别

  1. 编程式事务
  2. 声明式事务：建立在AOP上，本质是通过AOP功能，对方法前后进行拦截，将事务处理的功能编织到拦截的方法中，即在目标方法开始之前加入一个事务，在执行完之后根据执行情况提交或回滚事务

声明式事务最大优点是不需要在业务逻辑代码中掺杂事务管理的代码，只需在配置文件中做相关的事务规则声明或通过@Transactional注解的方式，将事务规则应用到业务逻辑中。是倡导使用的非侵入式开发方式，使业务代码不受污染。但只能作用到方法级别，而编程式事务可以作用到代码块级别

BeanFactory和FactoryBean的区别

  1. BeanFactory是一个工厂类（接口），负责生产和管理bean的一个工厂，通过它获取工厂管理的对象，是Spring IoC容器的核心接口，包括实例化、定位、配置应用程序中的对象和建立依赖。
    使用场景：1）从IoC容器获取Bean（byName, byType)；2）检索IoC容器中是否包含指定的Bean；3）判断Bean是否为单例
  2. FactoryBean是一个Bean，实现了FactoryBean接口，用户可以通过实现该接口定制实例化Bean的逻辑。优化了依赖对象装配到Spring容器中的过程，隐藏了实例化一些复杂Bean的细节。FactoryBean表现的是一个工厂的职责，调用实际获取的是工厂调用getObject()返回的对象，而不是Bean本身
    用法：1）类有很多属性时，简化xml配置，隐藏细节；2）返回不同Bean的实例
    使用场景：创建AOP的代理对象

Spring MVC工作原理

  1. 客户端发送请求到DispatcherServlet
  2. DispatcherServlet查询handlerMapping找到处理请求的Controller
  3. Controller调用业务逻辑后，返回ModelAndView
  4. DIspatcherServlet查询ModelAndView找到指定视图
  5. 视图将结果返回到客户端

Spring MVC执行流程及工作原理

  1. 用户发送请求至前端控制器DispatcherServlet
  2. DIspatcherServlet收到请求调用处理器映射器HandlerMapping
  3. 处理器映射器根据请求url找到具体的处理器，生成处理器执行链HandlerExecutionChain（包括处理器对象和处理器拦截器）一并返回给DispatcherServlet
  4. DIspatcherServlet根据处理器Handler获取处理器适配器HandlerAdapter执行HandlerAdapter处理一系列操作
  5. 执行处理器Handler（Controller，页面控制器）
  6. Handler执行完成返回ModelAndView
  7. HandlerAdapter将Handler执行结果ModelAndView返回给DispatcherServlet
  8. DispatcherServlet将ModelAndView传给ViewReslover视图解析器
  9. ViewReslover解析后返回具体View
  10. DispatcherServlet对View进行渲染视图（将模型数据填充至视图中）
  11. DispatcherServlet响应用户

Spring MVC中如何解决GET|POST请求中文乱码问题

GET：

  1. 每次请求前对URL编码
  2. 修改tomcat配置文件server.xml增加Encoding=”utf-8”

POST：

  1. request解析数据时设置编码格式
  2. 使用编码过滤器解决，在Web.xml文件增添加过滤器配置

Spring MVC如何设置重定向和转发

  1. 重定向：redirect:
  2. 转发：forward:

Spring MVC如何处理异常

  1. 异常抛给Spring框架，由框架处理
  2. 自定义实现Spring的全局异常解析器HandlerExceptionResolver，在异常处理器中添加视图页面

Spring拦截器如何使用

  1. 定义拦截器，实现HandlerInterceptor接口。三个方法：1）preHandler：进入Handler方法之前执行，用于身份认证、身份授权；2）postHandler：进入Handler之后，返回ModelAndView之前执行；3）afterCompletion：执行Handler完成执行此方法
  2. 拦截器配置：1）针对HandlerMapping配置（不推荐）；2）类似全局的拦截器，注入到每个HandlerMapping，继承适配器类，然后在接口方法中实现处理逻辑，在配置文件中配置拦截器

SpringMVC的控制器是不是单例模式？有什么问题？如何解决

是单例模式，在多线程访问时有线程安全问题，解决方案是在控制器里不能写可变状态量，如果要使用可以使用ThreadLocal，为每个线程单独生成一份变量副本

怎么把ModelMap的数据放入Session里

在类上面加@SessionAttributes注解，里面包含的字符串就是要放入session的key

SpringBoot

starter之间的关系

spring-boot-starter-parent是一个特殊的启动器，提供有用的maven默认值，还提供了依赖管理的部分，使得可以省略version标签带来的限制的依赖关系。其他starter仅提供开发特定类型的应用程序时可能需要的依赖项

SpringBoot应用程序启动器，均以spring-boot-开头
  1. starter：核心starter工具，包括自动配置支持、日志记录和yaml
  2. starter-activemq：使用Apache ActiveMQ的JMS消息传递Starter
  3. starter-amqp：使用Spring AMQP和RabbitMQ的starter
  4. starter-aop：使用Spring AOP和AspectJ进行面向方面编程的starter
  5. starter-data-mongodb：使用MongoDB面向文档的数据库和Spring Data MongoDB的starter
  6. starter-data-redis：将Redis键值数据存储与Spring Data Redis和JedisClient端一起使用的Starter

  8. starter-mail：开始使用Java Mail和Spring框架的电子邮件发送支持
  9. starter-security：使用Spring Security的starter
  10. starter-test：用于使用包括JUnit在内的库测试Spring Boot应用程序的starter
  11. starter-web：使用Spring MVC构建Web应用程序的starter，Tomcat是默认的嵌入式容器
  12. starter-web-services：使用Spring Web services
  13. starter-websocket：使用Spring框架的WebSocket支持构建WebSocket应用程序
SpringBoot生产程序启动器
  1. starter-actuator：使用SpringBoot的Actuator的starter程序，提供生产就绪功能，帮助监视和管理应用程序
SpringBoot技术启动器
  1. starter-log4j2：使用Log4j2进行日志记录
  2. starter-logging：使用Logback进行日志记录，默认记录启动器

SpringBoot哪些优点

快速开发，快速整合，配置简化，内嵌服务容器

JavaConfig是什么

提供了配置Spring IoC容器的纯Java方法，有助于避免使用XML配置

  1. 面向对象的配置
  2. 减少或消除XML配置
  3. 类型安全和重构友好

如何无需重启服务器加载SpringBoot上的更改

使用DEV工具。添加依赖spring-boot-devtools，可以自动部署文件更改

SpringBoot的监视器

spring boot actuator监视器可以帮助访问生产环境中正在运行的应用程序的当前状态

Spring Profiles

允许用户根据配置文件（dev，test，prod）注册bean，当应用程序在开发中运行时，只有某些bean可以加载，而在prod中其他bean可以加载

CSRF攻击

跨站请求伪造，迫使最终用户在当前通过身份验证的Web应用程序上执行不需要的操作，CSRF攻击专门针对状态改变请求，而不是数据窃取，因为攻击者无法查看对伪造请求的响应

什么是WebSockets

WebSocket是一种计算机通信协议，通过单个TCP连接提供全双工通信信道

  1. WebSocket是双向的：使用WebSocket客户端或服务器可以发起消息发送
  2. WebSocket是全双工的：客户端与服务器通信是相互独立的
  3. 单个TCP连接：初始连接使用http，然后将连接升级到基于套接字的连接，单一连接用于未来所有通信
  4. 轻量级：与http相比，websocket消息数据交换要轻很多

SpringBoot核心注解

@SpringBootApplication，包含以下3个注解

  1. @SpringBootConfiguration：组合了@Configuration注解，实现配置文件的功能
  2. @EnableAutoConfiguration：打开自动配置的功能，也可以关闭某个自动配置的选项
  3. @ComponentScan：Spring组件扫描

SpringBoot支持哪些日志框架，推荐和默认的日志框架

支持Java Util Logging, Log4j2, Logback。默认为LogBack

SpringBoot的自动配置原理

@EnableAutoConfiguration

  1. 从配置文件META_INF/Spring.factories加载可能用到的自动配置类
  2. 去重，将exclude和excludeName属性携带的类排除
  3. 过滤，将满足条件的自动配置类返回

SpringBoot配置加载顺序

  1. .properties文件
  2. .yaml文件
  3. 系统环境变量
  4. 命令行参数

SpringBoot核心配置文件是什么？bootstrap.properties和application.properties有何区别

  1. bootstrap(.yml, .properties)：bootstrap由父ApplicationContext加载，比application优先加载，配置在应用程序上下文的引导阶段生效，一般在Spring Cloud配置使用
  2. application(.yml, .properties)：由ApplicationContext加载，用于SpringBoot项目的自动化配置

比较Spring Security和Shiro各自优缺点

  1. Spring Security是一个重量级的安全管理框架，只需要添加一个依赖就可以保护所有的接口，Shiro是一个轻量级的安全管理框架
  2. Spring Security概念复杂，配置繁琐，Shiro概念简单，配置简单
  3. Spring Security功能强大，Shiro功能简单

SpringBoot如何解决跨域问题

通过CORS（Cross-origin resource sharing）跨域资源共享解决，实现WebMvcConfigure接口然后重写addCorsMappings方法

SpringBoot性能如何优化

  1. 项目比较大，类比较多，不适用@SpringBootApplication，采用@Component指定扫包范围
  2. 在项目启动时设置JVM初始内存和最大内存相同

SpringBoot打包的jar和普通jar有什么区别

SpringBoot项目最终打包的jar是可执行jar，通过java -jar xx.jar运行，不可以作为普通jar被其他项目依赖，主要是因为结构不同，打包的jar解压后\BOOT-INF\classes目录下才是代码，而普通jar解压后就是代码。

缓存

分为一级缓存和二级缓存，默认一级缓存开启且不能关闭。

一级缓存

SqlSession级别的缓存，在同一个SqlSession中进行相同的SQL语句查询时，第二次以后的查询不会从数据库查询，而是直接从缓存中获取，一级缓存最多缓存1024条SQL
第一次查询sql，将结果写入sqlsession的一级缓存，使用数据结构是map
key: MapperId + offset + limit + sql + 所有入参
value：用户信息
后续同一个sqlsession发出相同sql，可以从缓存中取数据。如果中间出现commit操作（更新），则本sqlsession的一级缓存全部清空，下次需要再从数据库查询写入缓存

二级缓存

跨SqlSession的缓存，是mapper级别的，在不同的SqlSession之间可以共享
范围是mapper级别，以命名空间为单位创建缓存数据结构，也是map。通过CacheExecutor实现
key: MapperId + offset + limit + sql + 所有入参
