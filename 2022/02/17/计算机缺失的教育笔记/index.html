变量

特殊参数变量

  1. $0 脚本名
  2. $1-9 传入脚本的参数
  3. $@ 所有参数
  4. $# 参数个数
  5. $? 前一个命令返回值
  6. $$ 当前脚本进程码

Vim常用命令

移动

  1. 基本移动：hjkl
  2. 词：w下一个单词，b词初，e词尾
  3. 行：0行初，^第一个非空格字符，$行尾
  4. 屏幕：H首行，M中间，L底部
  5. 翻页：ctrl-u上翻，ctrl-d下翻
  6. 文件：gg文件头，G文件尾

在进入可视化模式后，可以通过移动命令选中，对选中的区域整体操作，比如复制修改等

O/o，在上/下插入行

x：删除光标字符，等价于delete
s：替换，等于x+i
u：撤销，ctrl+r重做
p：粘贴

命令行环境

任务控制

shell使用信号机制执行进程间通信，信号是软件中断
输入ctrl+C，shell发送SIGINT信号到进程
ctrl+\发送SIGQUIT信号
ctrl+Z发送SIGTSTP暂停进程，进程会进入suspend状态
SIGSTOP同样暂停进程，区别是SIGSTOP不可捕获
fg或bg恢复暂停的工作，分别在前台和后台。
jobs列出当前终端会话尚未完成的全部任务
&后缀让命令直接在后台运行，但会使用shell的标准输出。但会随着终端关闭（发送SIGHUP信号）而停止。可以使用nohup封装命令忽略SIGHUP信号
应用示例

终端多路复用

在一个终端窗口中分割出多个终端，以遍同时与多个shell会话交互。（想想自己之前打开多个终端的行为简直笨死）

tmux

终端多路器，快捷键，都是Ctrl+b x的组合，即先按ctrl+b，再按特定指令

会话

tmux 开始一个新会话
tmux new -s NAME 指定名称开始新会话
tmux ls 列出当前所有会话
ctrl+b d，分离当前会话，切回到原始的bash界面
tmux a 重连最后一个会话
tmux attach -t 0 附加到某个会话，0是tmux ls的输出
tmux rename-session -t 0 database 重命名现有会话

窗口

ctrl+b c 创建新窗口
ctrl+b N 跳转到第N个窗口
ctrl+b p 切换到前一个
ctrl+b n 切换到下一个
ctrl+b , 重命名
ctrl+b w 列出当前所有窗口。每个窗口包括N个面板

面板

即分屏，在一个屏幕里显示多个shell
ctrl+b “ 水平分割
ctrl+b % 垂直分割，一个屏幕监控多个资源状态挺方便
ctrl+b <方向> 切换面板
ctrl+b z 切换当前面板缩放，将当前面板放大至全屏或恢复原状
ctrl+b [ 往回卷动屏幕，按下空格选择，回车复制选中部分。试了下么看懂，esc返回
ctrl+b 空格，在不同面板间切换，尝试之后发现会自动改变当前多个面板排版，比如T型变成三个连续的竖型。

转载自tmux教程，至少看起来非常高大上
安装：sudo apt-get install tmux或者brew install tmux(mac)

日志

日志的优势

  1. 可以写入文件、socket甚至服务器。这点有体会，做MIT6.824 Lab时日志在便准输出显示不完整，无法做深入分析。
  2. 支持严重等级，INFO等，可以过滤
  3. 新问题，在日志中包含了定位的信息，不需要重新运行复现问题

调试

调试器功能

  1. 到达某一行时暂停程序
  2. 依次一条指令执行
  3. 程序崩溃后查看变量的值
  4. 满足特定条件时暂停程序

python调试器pdb
l(ist) - 显示当前行附近的11行或继续执行之前的显示；
s(tep) - 执行当前行，并在第一个可能的地方停止；
n(ext) - 继续执行直到当前函数的下一条语句或者 return 语句；
b(reak) - 设置断点（基于传入的参数），在某行设置断点；
p(rint) - 在当前上下文对表达式求值并打印结果。还有一个命令是pp ，它使用 pprint 打印；
r(eturn) - 继续执行直到当前函数返回；
q(uit) - 退出调试器。
cl(ear) - 删除断点，参数是调用无参的b指令输出的断点列表中的序号
! - !命令指明后续语句是python命令而不是pdb

ipdb是增强的pdb，但貌似和py2兼容性不太好，暂时略过

性能分析

此前常被忽视的一个点，直到做了北大项目才理解用途。用来检验某段程序能否在限定时间内完成。
time sh 可以显示多种时间
real 真实时间：程序从开始到结束花费的时间
user 用户时间，执行用户代码的时长
sys 系统时间，执行系统内核代码的时间

通常是CPU性能分析工具。追踪分析器记录程序的每次函数调用，采样分析器周期性的监测程序并记录对堆栈。最终生成统计信息，显示花费时间的分布。

python 使用cProfile模块分析每次函数调用消耗的时间，会显示每次函数调用的时间

添加@profile装饰器并使用对应分析工具

依赖管理

版本号，通常格式为主版本号.次版本号.补丁号，规则：

  1. 如果新版本没改变api，则补丁号递增
  2. 如果添加了api且改动向后兼容，将次版本号递增
  3. 如果修改了api且不向后兼容，则主版本号递增

此时，基于本项目构建的所有项目，只要最新版本的主版本号没变就安全，次版本号不低于项目构建时的版本即可。

持续集成

也叫CI，指的是“当代码发生变动时，自动运行的东西”。比如再仓库中添加一个文件，CI系统启动一个虚拟机，执行既定规则，并记录执行结果。比如在git提交代码到远程仓库后，CI会启动虚拟机执行测试验证本次提交没有影响此前的功能。

github创建提交后的操作

在Github选中仓库后，进入Actions设置，修改main.yml并提交。可以参考右侧的功能或者进入Github MarkerPlace选择想实现的功能，复制对应的配置。
效果就是在每次提交后会自动执行对应的CI检查。每个yml对应一个检查。

安全和密码学

散列函数

将任意大小的数据映射为一个固定大小的输出。
hash(value: array) -> vector<byte, N>，N固定
SHA1在git中使用，映射为40位十六进制。
散列函数不可逆，且看上去随机但具有确定性的函数。散列函数的特性：

  1. 确定性：同样输入有相同输出
  2. 不可逆性：很难根据散列值计算出原始值
  3. 弱无碰撞：给定输入，很难找到一个与给定输入不同但散列值相同的对象
  4. 强无碰撞：难以找到一组输入得到同样的散列值

散列函数应用

  1. git的内容寻址
  2. 文件信息摘要，比如确认下载的文件是否与官方一致
  3. 承诺机制：希望承诺一个值，但在之后透露他，可以不提前泄露信息又能确保没作弊

对称加密

keygen() -> key (这是一个随机方法)

encrypt(plaintext: array, key) -> array (输出密文)
decrypt(ciphertext: array, key) -> array (输出明文)

AES是常用的对称加密系统
例如 加密不信任的云服务上存储的文件

非对称加密

使用公钥和私钥对。
keygen() -> (public key, private key) (这是一个随机方法)

encrypt(plaintext: array, public key) -> array (输出密文)
decrypt(ciphertext: array, private key) -> array (输出明文)

sign(message: array, private key) -> array (生成签名)
verify(message: array, signature: array, public key) -> bool (验证签名是否是由和这个公钥相关的私钥生成的)

信息在非对称加密中使用 公钥 加密， 且输出的密文很难在不知道 私钥 的情况下得出明文。
解密方法decrypt()有明显的正确性。 给定密文及私钥，解密方法一定会输出明文： decrypt(encrypt(m, public key), private key) = m

应用：

  1. 电子邮件加密
  2. 聊天加密
  3. 软件签名
