你的缺点

要说出具体缺点，但不会很大影响工作，并且要强调自己是如何克服这个缺点。
例如：“有时候，我对细节不够重视。好的一面是我反应迅速，执行力强，但不免会因为
粗心大意而犯错。有鉴于此，我总是会找其他同事帮忙检查自己的工作，确保不出问题。”

我的缺点：也是不够重视细节，但我会和其他同学交流以确认自己的程序没有问题。

提问面试官的问题

  1. 真实的问题：关于公司日常工作情况的问题：1.团队中人员比例是多少，如何互动，团队怎么做项目规划，团队之间的交流方式
  2. 有见地的问题：1.我注意到你们使用X技术，请问是如何解决Y问题的。 需要对公司做充分调研，才问的出有深度的问题
  3. 有激情的问题，表现对技术的热忱。1.我对可扩展性很感兴趣，有哪些机会可以学习这方面知识？2.我对X技术不是很熟悉，您能讲一下他的工作原理吗？

王牌项目

主攻两三个项目，熟练掌握其中涉及的技术。应符合如下标准

  1. 有挑战性（不仅仅学到很多）
  2. 你是主力，最好负责具有挑战性的部分
  3. 能畅谈技术部分

你应当能够畅谈在王牌项目及其余项目中遇到的挑战、犯的错误、做出的技术决策、技术
选型中的取舍以及本可以做得更好的地方

应对行为面试题的建议

  1. 力求具体，切忌自大
  2. 省略细枝末节。只谈重点，说明效果
  3. 多谈自己，使用我作为主语。
  4. 回答条理清晰：方法有：1.主题先行法。直奔主题，让面试官了解事情梗概。2.SAR法：描述情景，解释行动，最后陈述结果
  5. 行动是关键
  6. 故事的意义

自我介绍

  1. 结构：按照时间顺序：目前的工作–大学时期–业余时间–总结
  2. 兴趣爱好是为了缓和气氛，是否谈论看情况。
  3. 展示成功的点点滴滴

准备事项

  1. 尽量独立解题
  2. 尝试在纸上写代码
  3. 在纸上测试代码
  4. 将代码照原样输入计算机
  5. 多做模拟面试

解题步骤

听–
举例（尽量特殊且小）–
蛮力法（不写代码）–
优化（BUD优化，也可以根据：1.寻找未利用信息；2.解决例题逆向思考；3.思考不正确解法为什么失败；4.权衡时间空间）–
梳理（回顾最有算法，理顺每个细节）–
实现（注意模块化和重构）–
测试（1.概念测试；2.异常代码；3.热点代码；4.小测试用例；5.特殊或边缘情况），发现错误要小心修复

优化和解题技巧：

  1. 寻找BUD：B（瓶颈，bottleneck），U（无用功，unnecessary work），D（重复工作，duplicated work）。
  2. 亲历亲为：设计一个真实场景并尝试通过直觉解决，而不是一个算法题
  3. 化繁为简：简化或调整约束，然后调整算法适应更复杂情况。
  4. 由浅入深：从n=1基本情况开始，然后构建更复杂的情况
  5. 数据结构头脑风暴：简单过一遍数据结构，尝试哪一个更有效。
  6. 可想像的极限运行时间BCR，对于某问题，能想象出的运行时间的最优极限，是无法超越的。然后在现有复杂度基础上进行优化，尝试如何达到最优解。例如知道BCR为O(N)，则对于当前O(N^2)解法则是将一层O(N)优化为O(1)。

处理错误答案

面试不要求能完美做出所有题目，关键在于思考过程以及优化过程。体现出的解决和优化难题的激情也是重要因素。

好代码的标准

  1. 正确
  2. 高效
  3. 简洁
  4. 可读性
  5. 可维护性
  6. 多用数据结构
  7. 适当代码复用
  8. 模块化
  9. 灵活性和通用性
  10. 错误检查

可以不写完整的代码，通过简写定义、假设存在某结构的方式，可以说明在有时间时补上，不会影响代码可读性。
一个加分项：表现出解决难题的满腔热情

录用通知和其他注意事项

处理录用与被拒的情况

  1. 录用后回复期限可以请求延长。
  2. 拒绝录用通知，要给出一个合乎情理且不容置疑的理由。
  3. 被拒之后可以视为一次重新申请的机会。

评估录用待遇

  1. 薪酬考量：奖金等津贴、年终奖和股票期权和补助金，可以除以签约年份算平均工资。
  2. 职业发展：主要是对职业发展又怎样的影响，这比薪资更重要。
  3. 公司稳定性：稳定的公司裁员几率小，但创业公司财富自由概率大。
  4. 幸福指数：要从产品、团队、企业文化和工作时长来综合考虑。

录用谈判

  1. 理直气壮
  2. 最好手头有其他选择
  3. 提出具体要价而不是泛泛
  4. 开出比预期稍高的价码
  5. 不要只在乎薪水
  6. 使用最合适的方法

入职须知

  1. 指定时间表：入职只是职业的起点。不要得意于当前找到了工作，应该做好职业规划，想想十年后想做什么？达成目标要怎么一步步实现？每年总结下自己在职业和技能上有哪些进步，明年有什么规划？
  2. 打造坚实的人际网络：工作中要和同事建立良好的关系。说不定在哪里就会有需要帮助的地方，当需要帮助别人时也要出手相助。
  3. 向经理求助：和主管沟通，想做技术就多争取项目，做管理就探讨需要做哪些准备
  4. 保持面试状态：不一定要真的换工作，但可以提高面试技能

BFS求解问题可以考虑使用双向搜索的方法，速度可以加快O(n^(k/2))倍。

位操作

逻辑右移使用>>>表示，连同符号位一起移动，在高位补0，算数右移使用>>表示，只移动数字位，并在高位补符号位。

常见位操作:

  1. 获取数位；得到num第i位的值
  2. 设置数位：将第i位设为1
  3. 清零数位：将第i位设置为0 清零最高位至第i位所有的数位 清零第i位至第0位的所有数位
  4. 更新第i位值为v
